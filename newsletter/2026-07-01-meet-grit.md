## The first month is rarely about your product

Every full-stack project starts the same way. Before you write the feature that
actually matters, you re-solve the problems you already solved on the last three
projects: auth, an admin panel, file uploads, background jobs, email, rate
limiting, a type-safe API client. None of it is hard. All of it is slow. And none
of it makes your product different.

**Grit** exists to delete that month.

## What Grit is

Grit is a full-stack meta-framework that fuses a **Go** backend (Gin + GORM) with
a **React** frontend (Next.js or Vite + TanStack Router) and a Filament-style
**admin panel** — in a single monorepo with shared types. One command scaffolds
all of it, wired together and hardened for production:

```bash
grit new my-app --triple
```

That gives you a Go API, a customer-facing web app, and an admin dashboard that
already share the same types and validation. No glue code.

## The part that feels like magic

The bigger idea is the generator. Describe a resource, and Grit writes the whole
vertical slice — backend *and* frontend — in one go:

```bash
grit generate resource Product --fields "name:string,price:float,stock:int"
```

Out comes the Go model, service, and handler; the routes; the Zod schema and
TypeScript types; the React Query hooks; and an admin page — all consistent, all
type-safe, all following the same conventions. You extend clean code instead of
gluing libraries.

## Batteries, not a starter kit

A starter kit gives you a foundation and a to-do list. Grit ships the features
already working on day one:

- **Auth** — JWT, OAuth2 social login, 2FA (TOTP + backup codes)
- **File storage** — S3 / R2 / MinIO with presigned uploads
- **Background jobs + cron** — Redis-backed queue
- **Email** — transactional templates (Resend, Mailhog in dev)
- **AI gateway** — hundreds of models behind one API key
- **Observability** — request tracing, DB monitoring, metrics
- **Security** — OWASP-2025 patterns, rate limiting, audit logging
- **A visual DB browser** at `/studio`

## Who it's for

- **Go developers** who want a batteries-included framework without giving up Go.
- **Full-stack builders** who want Rails/Laravel-style DX with Go's performance.
- **Teams and solo founders** who'd rather ship product than assemble plumbing.
- **AI agents** — Grit's strong conventions mean thousands of decisions an
  assistant doesn't have to make, so generated code stays clean and extendable.

## Try it

Install the CLI and scaffold your first app in about five minutes:

```bash
# macOS / Linux
curl -fsSL https://gritframework.dev/install.sh | sh

# Windows (PowerShell)
iwr -useb https://gritframework.dev/install.ps1 | iex
```

Tomorrow in **The Daily Grit**: your first Grit app, running end-to-end, in five
minutes flat.
